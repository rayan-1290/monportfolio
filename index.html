Documentation des compétences — Projet Seyrin

C1

Recenser et identifier les ressources numériques

Réalisation : Recensement d'un parc informatique avec GLPI dans le cadre du projet Seyrin.

Contexte

Dans le cadre du Hackathon, nous avons mis en place un outil de gestion de parc informatique afin de centraliser les informations des équipements et des utilisateurs. L'objectif était d'avoir une vision claire du parc informatique grâce à GLPI.

Démarche suivie

  • Installation de GLPI sur un serveur Linux avec les dépendances nécessaires (Apache, PHP, MariaDB)
  • Création de la base de données dédiée et configuration des accès
  • Finalisation de l'installation via l'interface web de GLPI
  • Installation et configuration de l'agent GLPI sur les postes pour remonter automatiquement les informations vers le serveur
  • Création des comptes utilisateurs pour chaque membre de l'organisation
  • Attribution des rôles à chaque compte (administrateur, technicien, utilisateur) pour gérer les droits d'accès
  • Organisation du parc : identification des machines, de leurs caractéristiques et structuration des ressources dans l'outil
# Installation des dépendances
$ sudo apt install apache2 php php-mysql php-curl mariadb-server
 
# Installation et configuration de l'agent GLPI
$ sudo apt install glpi-agent
$ sudo nano /etc/glpi-agent/agent.cfg
server = http://IP_SERVEUR/glpi
$ sudo systemctl enable glpi-agent && sudo systemctl start glpi-agent

Compétences acquises

  • Installer et configurer GLPI ainsi que son agent sur plusieurs machines
  • Recenser et identifier les ressources numériques d'une organisation
  • Structurer un parc informatique en attribuant des rôles et en centralisant les informations
  • Gérer les droits d'accès et les responsabilités au sein d'un outil de gestion de parc
C11

Référencement des services en ligne et mesure de leur visibilité

Réalisation : Mise en place de Google Analytics sur le site vitrine Seyrin Agency hébergé sur GitHub Pages.

Contexte

Dans le cadre du projet Hackathon, notre équipe Seyrin disposait d'un site vitrine hébergé sur GitHub Pages. L'objectif était de mettre en place un outil de mesure pour évaluer la visibilité du site en ligne et analyser le comportement des visiteurs grâce à Google Analytics.

Démarche suivie

  • Création d'un compte Google Analytics et d'une propriété pour le site vitrine de Seyrin Agency
  • Configuration du flux de données web avec l'URL du site GitHub Pages et récupération de l'identifiant de mesure (format G-XXXXXXXXXX)
  • Intégration du script de suivi Google Analytics (gtag.js) dans le <head> de chaque page du site vitrine
  • Vérification du bon fonctionnement : accès depuis plusieurs navigateurs et appareils, contrôle en temps réel dans Google Analytics, vérification de l'absence d'erreurs en console
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

Indicateurs analysés

  • Nombre de visiteurs sur une période donnée
  • Pages les plus consultées (Accueil, Équipe, Documentation)
  • Sources de trafic : accès direct, moteur de recherche, réseaux sociaux
  • Durée moyenne des sessions et taux de rebond

Compétences acquises

  • Créer et configurer une propriété Google Analytics
  • Intégrer un script de suivi sur un site statique (GitHub Pages)
  • Analyser les indicateurs de fréquentation et de visibilité d'un site web
  • Prendre des décisions d'amélioration basées sur des données concrètes
C7

Mettre à disposition des utilisateurs un service informatique

Réalisation : Déploiement et mise à disposition du service GLPI et du site vitrine pour l'ensemble de l'équipe Seyrin.

Contexte

Le projet Seyrin nécessitait la mise à disposition de deux services principaux : un site vitrine pour la présentation de l'agence et une instance GLPI pour la gestion des tickets et du parc informatique. Ces services devaient être accessibles à distance par tous les membres de l'équipe, quel que soit leur emplacement géographique.

Démarche suivie

  • Installation et configuration du serveur web Apache sur la machine virtuelle Azure (Ubuntu Server)
  • Déploiement de l'application GLPI avec les modules PHP nécessaires (php-mysql, php-curl, php-gd, php-xml, php-mbstring)
  • Transfert des fichiers du site vitrine et de GLPI vers le répertoire /var/www/html/ du serveur via SCP
  • Configuration des règles de sécurité réseau (NSG Azure) pour autoriser le trafic HTTP (port 80) et HTTPS (port 443)
  • Tests d'accessibilité depuis des postes externes (connexion 4G, réseaux Wi-Fi tiers) pour valider la disponibilité des services
  • Rédaction d'une documentation utilisateur expliquant l'accès aux services et les identifiants de connexion

Résultats

Les deux services ont été rendus accessibles de façon stable et sécurisée à l'ensemble de l'équipe. Chaque membre pouvait se connecter à GLPI pour créer et suivre des tickets, et consulter le site vitrine depuis n'importe quel navigateur. La documentation fournie a permis une prise en main rapide par tous les utilisateurs.

Compétences acquises

  • Installer et configurer un serveur web pour héberger des applications
  • Gérer les droits d'accès et les règles réseau pour sécuriser un service
  • Valider la disponibilité d'un service depuis différents environnements
  • Rédiger une documentation de mise à disposition à destination des utilisateurs

C17

Déployer et rendre accessible un service informatique en ligne

Réalisation : Déploiement complet de l'infrastructure cloud sur Microsoft Azure — hébergement du site et déploiement sur Azure.

Contexte

Pour rendre le site vitrine et l'instance GLPI du projet Seyrin accessibles depuis l'extérieur, il était nécessaire de déployer l'ensemble de l'infrastructure sur le cloud. En tant que membre de l'équipe Infrastructure, j'ai participé à l'hébergement du site et au déploiement complet sur Microsoft Azure.

Démarche suivie

  • Création d'un groupe de ressources Azure pour regrouper les composants du projet (VM, IP, réseau virtuel, disque)
  • Déploiement d'une machine virtuelle Linux (Ubuntu Server) avec une taille adaptée aux besoins et une région géographique optimisée
  • Configuration du Network Security Group (NSG) : ouverture des ports 22 (SSH), 80 (HTTP) et 443 (HTTPS)
  • Association d'une adresse IP publique statique à la VM
  • Installation d'Apache et des modules PHP requis sur la VM
  • Transfert des fichiers du site et de GLPI via SCP
  • Configuration DNS : enregistrements A pointant vers l'IP publique Azure
  • Installation de Certbot et génération d'un certificat SSL Let's Encrypt pour sécuriser les communications en HTTPS
  • Configuration du VirtualHost Apache avec les directives SSL
# Déploiement VM Azure
$ az vm open-port --port 80 --resource-group MonGroupe --name MaVM
$ az vm open-port --port 443 --resource-group MonGroupe --name MaVM
 
# Installation serveur web
$ sudo apt install -y apache2 php php-mysql php-curl php-gd php-xml
 
# Sécurisation HTTPS
$ sudo certbot --apache -d monsite.fr -d glpi.monsite.fr

Tests et validation

  • Validation DNS avec nslookup et dig, vérification de la propagation mondiale
  • Test du certificat SSL dans le navigateur (cadenas valide, pas d'avertissement)
  • Test de renouvellement automatique : sudo certbot renew --dry-run
  • Accès depuis des postes externes (4G, Wi-Fi tiers) pour confirmer la disponibilité

Compétences acquises

  • Déployer et configurer une infrastructure cloud sur Microsoft Azure
  • Gérer les groupes de ressources, les règles réseau et les IP publiques Azure
  • Installer et configurer un serveur web Apache en environnement de production
  • Mettre en œuvre le protocole HTTPS avec Let's Encrypt et Certbot
  • Configurer et diagnostiquer les enregistrements DNS
